üîç The Proxy

This is the heart of Burp Suite. It allows you to intercept, inspect, and also modify HTTP/S requests and responses between your browser and the target application.

How to Use It:
  • Toggle interception on/off in "Proxy" > "Intercept".
  • Use "HTTP history" to view past requests.
  • Modify parameters, headers, or cookies in real-time.
Use case example: Intercept a login POST request ‚Üí modify parameters ‚Üí forward ‚Üí observe response.
üîÅ The Repeater

Repeater allows you to manually modify and resend individual HTTP requests to observe how the server responds. This is useful for testing different payloads or edge cases.

How to Use It:
  • Right-click any request in Proxy or HTTP history > ‚ÄúSend to Repeater‚Äù
  • Modify the request body, headers, or method (GET, POST, etc.)
  • Click "Send" and view the raw response.
Use case example: Testing different SQL injection payloads on a search form.

üóÇÔ∏è Getting Started with Burp Suite

‚öôÔ∏è Proxy Configuration

In order to get your browser working with Burp and intercept traffic, it acts as a local proxy. The default address is: 127.0.0.1:8080.

Important considerations:
  • Understand the role of Burp's Proxy tool in intercepting and modifying HTTP/S requests.
  • Configure your browser to use Burp as a proxy or use the included version of Chromium.
  • Learn to intercept and manipulate requests using Burp Proxy.
Steps to connect:
  1. Launch Burp and go to the "Proxy" > "Options" tab.
  2. Confirm the proxy listener is active on port 8080.
  3. Configure your browser (e.g., Firefox) to use 127.0.0.1:8080 as the HTTP and HTTPS proxy.
  4. Import Burp‚Äôs certificate into the browser to prevent SSL errors (via `http://burp` in the browser).
